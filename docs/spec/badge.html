Component · Badge

Badge.

Three components, not one. StatusBadge carries safety ratings and is loud. CategoryTag labels categories and stays quiet. AccentBadge marks editorial moments on amethyst surfaces only. Each has its own shape so visual weight maps to role — none of them are interactive.

Amethyst surfaces only

Lime on cream or paper fails contrast, and the design system pivots on lime being the signature accent-on-amethyst pairing. If the surface isn't amethyst, reach for StatusBadge or CategoryTag instead. One AccentBadge per screen, max — the whole point is that it's rare.

Not this

Clean
Don't — StatusBadge on -tint

Tint is too quiet. StatusBadge always uses the saturated -bg. Tint is reserved for larger callout panels that contain a badge.

Pills collide with StatusBadge shape. Status palette on categories implies category = safety. Neither should happen.

Accessibility
  • Not interactive. Badges have no focus, no hover, no press, no disabled state. If you need clickable, use a Button (ghost) or a filter chip.
  • StatusBadge: icons are aria-hidden="true". Label text is the announced name. For product context, wrap with aria-label="Rated Clean — Aesop body wash".
  • CategoryTag: label text is the announced name. If used as filter selection state, the parent element owns the interactive semantics.
  • AccentBadge: decorative by default. If it carries meaning, expose in parent aria-label or adjacent text.
  • All color pairs pass WCAG AA — most pass AAA. See badge.md for the full contrast matrix.
  • No motion. Respects prefers-reduced-motion by default (nothing to disable).
