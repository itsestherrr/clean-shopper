Component · Button

Button.

The commit control. Three variants (primary, secondary, ghost), three sizes, optional hero for one-per-surface big moments. Pill-shaped always — asymmetric radii are reserved for cards. Flat-offset shadow is the brand signature, but it's an opt-in hero treatment that only appears on hover, not a default button decoration.

States — hero

Rest No shadow at rest — same as non-hero
Hover Shadow appears + translate(-2px,-2px)
Press scale(0.96) · shadow dismisses
Why no resting shadow on hero?

The flat-offset amethyst shadow is the brand's loudest signal. If it's always on, it competes with card shadows across dense screens. Turning it into a response to hover keeps hero quiet at baseline and makes the interaction feel earned. Hero still communicates importance through layout position, size, and copy — not a resting shadow.

Accessibility
  • Native <button> — Tab, Enter, Space work for free
  • Focus ring: 2px slate outline, +2px offset, :focus-visible only (keyboard nav)
  • Loading: aria-busy="true", clicks suppressed, label remains as the announced name
  • Icons are decorative: wrap with aria-hidden="true"
  • Icon-only use case should not occur — route to IconButton which enforces aria-label
  • All color pairs pass WCAG AA (most pass AAA) — see button.md for the contrast matrix
  • Honors prefers-reduced-motion: drops scale + translate, uses instant color swaps
