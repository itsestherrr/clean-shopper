Component · IconButton

IconButton.

The workhorse control. Circular, icon-only, no signature shadow or motion of its own — saves the brand's loudest treatment for ProductCard and Button. Always requires aria-label because there is no visible text. Two variants, two sizes, an active state, and color-only transitions.

Why no shadow or press motion? (IB-01)

A ProductCard search grid renders 20+ save hearts at once. Adding a flat-offset shadow to IconButton would either compete with the card's hover signature, or stack lime-shadow-on-lime-shadow when a heart sits on a card. IconButton stays color-only so the system has a single workhorse that disappears into context.

Accessibility
  • ariaLabel is required — there is no visible text fallback
  • When toggling state, swap the label too ("Save to library" ↔ "Remove from library") — color-only is not enough
  • Native <button> — Enter and Space activate, Tab focuses
  • Focus ring matches the system — 2px amethyst outline at 2px offset, :focus-visible only
  • Disabled buttons drop out of tab order via the HTML disabled attribute
  • Don't nest two IconButtons without at least gap-xs (4px) between them — neighboring 28px circles need breathing room
